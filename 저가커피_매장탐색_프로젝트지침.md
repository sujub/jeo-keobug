# 저가커피 매장 탐색 서비스 — 프로젝트 지침

> 내 위치 기반으로 주변 저가 커피 매장을 찾아주는 서비스 개발을 위한 공식 프로젝트 지침서

---

## 1. 프로젝트 개요

### 목적
사용자의 현재 위치를 기반으로 반경 내 저가 커피 브랜드 매장을 실시간으로 탐색하고, 거리·영업시간·혼잡도 등 핵심 정보를 빠르게 제공하는 서비스를 구축한다.

### 핵심 가치
- **접근성**: 누구나 쉽고 빠르게 근처 저가 커피를 찾을 수 있어야 한다.
- **정확성**: 실시간 위치 데이터와 신뢰할 수 있는 지도 API를 활용해 오정보를 최소화한다.
- **단순함**: 불필요한 기능 없이 핵심 기능에 집중한다.

## 2. 대상 브랜드 정의

### 저가 커피 브랜드 기준
아메리카노 기준 단가 **3,000원 이하** 또는 동등한 가격대를 저가 커피로 정의한다.

| 브랜드 | 대표 메뉴 기준가 | 비고 |
|--------|----------------|------|
| 메가커피 | 1,500원~ | 전국 체인 |
| 컴포즈커피 | 1,500원~ | 전국 체인 |
| 빽다방 | 1,500원~ | 전국 체인 |
| 더벤티 | 2,000원~ | 전국 체인 |
| 매머드커피 | 2,000원~ | 전국 체인 |
| 하삼동커피 | 1,500원~ | 지역 체인 포함 |

> **확장 가능성**: 신규 저가 브랜드 추가를 위한 관리자 인터페이스를 별도 구성한다.

## 3. 기술 스택 결정

### 지도 API 선택 기준 및 권장안

| 항목 | 카카오맵 API | 네이버 지도 API | 구글 맵 API |
|------|------------|--------------|------------|
| 국내 데이터 정확도 | ★★★★★ | ★★★★★ | ★★★☆☆ |
| 무료 쿼리 한도 | 300,000건/일 | 200,000건/일 | 28,500건/월 |
| 한국어 지원 | 네이티브 | 네이티브 | 제한적 |
| 장소 검색 API | 지원 | 지원 | 지원 |
| 실시간 혼잡도 | 미지원 | 미지원 | 지원(Popular times) |
| 국내 매장 DB | 매우 풍부 | 풍부 | 보통 |
| 비용 | 무료 (한도 내) | 무료 (한도 내) | 유료 (한도 초과 시) |

**권장: 카카오맵 API** — 국내 매장 데이터 정확도, 무료 쿼리 한도, 한국어 지원 측면에서 최적.
네이버 지도 API를 보조 또는 대안으로 고려한다.

### 플랫폼별 기술 스택

**웹 (MVP)**
- Frontend: React (Vite) + TypeScript
- 지도: Kakao Maps JavaScript SDK
- 위치: Geolocation API (브라우저)
- 상태 관리: Zustand 또는 React Query
- 스타일: Tailwind CSS

**모바일 (2단계)**
- React Native 또는 Flutter
- 위치: expo-location (RN) / geolocator (Flutter)
- 동일 지도 API 활용

**백엔드 (선택적)**
- Node.js (Express) 또는 Next.js API Routes
- 브랜드 목록·매장 즐겨찾기 관리용
- DB: PostgreSQL 또는 Firebase Firestore

## 4. 핵심 기능 명세

### 4-1. 필수 기능 (MVP)

**F01 — 현재 위치 탐지**
- 브라우저/앱 Geolocation API로 사용자 좌표(위도·경도) 획득
- 위치 권한 거부 시 수동 주소 입력 대체 플로우 제공
- 위치 정확도: 반경 50m 이내

**F02 — 주변 매장 검색**
- 기본 탐색 반경: 500m / 1km / 2km (사용자 선택)
- 카카오 장소 검색 API (`keyword.json`) 활용
- 검색 키워드: 등록된 브랜드명 순차 조회
- 결과: 매장명, 주소, 거리, 전화번호, 카카오 지도 링크

**F03 — 지도 표시**
- 사용자 현위치 마커 표시 (구분되는 색상)
- 검색된 저가 커피 매장 핀 표시
- 핀 클릭 시 매장 기본 정보 말풍선 표시

**F04 — 목록 뷰**
- 거리순 정렬 (기본값)
- 브랜드별 필터
- 영업 중 필터 (영업시간 데이터 있는 경우)

### 4-2. 추가 기능 (v1.1 이후)

**F05 — 즐겨찾기**: 자주 가는 매장 북마크 (로컬 스토리지 또는 계정 연동)

**F06 — 길찾기 연동**: 카카오맵 앱/네이버 지도 앱으로 딥링크 길찾기

**F07 — 가격 정보**: 대표 메뉴(아메리카노 기준) 가격 표시 — 직접 수집·관리 필요

**F08 — 알림**: 특정 반경 내 저가 커피 매장 진입 시 알림 (모바일 전용)

## 5. API 설계

### 카카오 장소 검색 API 호출 예시

```
GET https://dapi.kakao.com/v2/local/search/keyword.json
  ?query=메가커피
  &x={경도}
  &y={위도}
  &radius=1000
  &sort=distance
  &category_group_code=CE7

Headers:
  Authorization: KakaoAK {REST_API_KEY}
```

### 응답 데이터 구조 (활용 필드)

```json
{
  "documents": [
    {
      "id": "12345678",
      "place_name": "메가커피 강남점",
      "address_name": "서울 강남구 ...",
      "road_address_name": "서울 강남구 ...",
      "phone": "02-1234-5678",
      "distance": "350",
      "place_url": "https://place.map.kakao.com/...",
      "x": "127.027621",
      "y": "37.497175"
    }
  ]
}
```

### 다중 브랜드 병렬 조회 전략

```
모든 브랜드에 대해 Promise.allSettled() 병렬 호출
→ 결과 통합 → 거리순 정렬 → 중복 제거 (같은 place_id)
→ 화면 렌더링
```

> **주의**: API 호출 횟수 최적화를 위해 5분 캐싱 적용 (동일 위치·동일 반경 조건)

## 6. UI/UX 설계 원칙

### 화면 구성
```
[상단 헤더]       — 앱 로고, 반경 선택 토글
[지도 영역]       — 전체 화면의 55%, 핀·마커 표시
[하단 슬라이드]   — 전체 화면의 45%, 매장 목록 카드
                    위로 스와이프 → 전체 목록 모드
                    아래로 스와이프 → 지도 전체 모드
```

### UX 핵심 원칙
1. **3초 룰**: 앱 실행 후 3초 이내 첫 매장 목록이 표시되어야 한다.
2. **오류 친화성**: API 실패·위치 거부 시 사용자가 이해할 수 있는 안내 메시지 제공.
3. **오프라인 대응**: 마지막 검색 결과를 로컬 캐시로 표시.
4. **한손 조작**: 핵심 버튼은 화면 하단 엄지 손가락 도달 범위에 배치.

## 8. 제약 조건 및 리스크 관리

### API 제약
| 제약 항목 | 내용 | 대응 방안 |
|----------|------|---------|
| 카카오 무료 한도 | 300,000건/일 | 캐싱, 불필요한 중복 호출 제거 |
| CORS (웹 환경) | 프론트엔드 직접 호출 시 발생 가능 | 프록시 서버 또는 서버사이드 라우팅 |
| 매장 데이터 부정확 | API 데이터 오류·폐업 정보 지연 | 사용자 신고 기능 추가 (v2) |

### 데이터 리스크
- 저가 커피 브랜드 확인은 수동 검증 필요 (API가 가격 정보를 제공하지 않음)
- 신규 브랜드 등장 시 주기적 업데이트 필요 (월 1회 검토)
- 폐업·이전 매장 데이터가 카카오 DB에 남아 있을 수 있음

### 개인정보 보의
- 사용자 위치 정보는 **서버에 저장하지 않는다** (클라이언트 사이드에서만 처리)
- API 호출 시 위치 좌표만 전달, 사용자 식별 정보 미포함
- 개인정보 처리방침 페이지 필수 제공

## 10. 팀 역할 및 협업 규칙

### 브랜치 전략 (Git Flow)
```
main        — 배포 브랜치 (직접 push 금지)
develop     — 통합 개발 브랜치
feature/*   — 기능 개발 브랜치 (예: feature/kakao-map-integration)
hotfix/*    — 긴급 버그 수정
```

### 커밋 메시지 규칙
```
feat: 새 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 포맷 변경 (기능 변화 없음)
refactor: 코드 리팩토링
test: 테스트 코드
chore: 빌드·설정 변경
```

### 코드 리뷰 기준
- PR은 최소 1인 이상 리뷰 후 merge
- API 키·인증 정보는 절대 코드에 하드코딩 금지 (환경변수 사용)
- 컴포넌트 단위 테스트 권장

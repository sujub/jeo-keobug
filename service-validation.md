# 서비스 오류 검증 규칙

> 이 파일은 저가커피 매장탐색 서비스의 정상 작동 여부를 스스로 판단하기 위한 기준이다.
> 코드 수정, 기능 구현, 배포 전 검토 시 아래 규칙을 순서대로 적용하고 결과를 보고한다.

---

## 판정 기준

| 판정 | 의미 |
|------|------|
| ✅ PASS | 정상 동작 확인 |
| ❌ FAIL | 즉시 수정 필요 — 서비스 비정상 |
| ⚠️ WARNING | 동작하지만 개선 권장 |

**종합 상태**:
- 정상: FAIL 0건
- 주의: FAIL 0건, WARNING 4건 이상
- 비정상: FAIL 1건 이상 → 배포 중단

---

## RULE-01 · 위치 탐지

**언제 적용**: Geolocation 관련 코드 수정 시, 위치 마커 컴포넌트 수정 시

| ID | 검증 내용 | FAIL 조건 |
|----|----------|----------|
| 01-A | 위치 권한 요청 팝업이 앱 실행 시 표시되는가 | 팝업 미표시 |
| 01-B | 권한 허용 후 지도가 현재 위치 중심으로 이동하는가 | 위치 이동 없음 |
| 01-C | 현재 위치 마커가 지도 위에 렌더링되는가 | 마커 미표시 |
| 01-D | 위치 권한 거부 시 수동 주소 입력 UI가 표시되는가 | 대체 UI 없음 |
| 01-E | 수동 주소 입력 후 해당 좌표로 지도가 이동하는가 | 이동 없음 |

**코드 검증 포인트**:
```
- navigator.geolocation.getCurrentPosition() 콜백에 에러 핸들러 존재 여부
- 위치 권한 거부(PermissionDenied) 분기 처리 존재 여부
- 마커 좌표 = API에서 받은 실제 위도·경도 (하드코딩 금지)
```

## RULE-02 · 브랜드 검색

**언제 적용**: 카카오 장소 검색 API 연동 코드 수정 시, 브랜드 목록 변경 시

| ID | 검증 내용 | FAIL 조건 |
|----|----------|----------|
| 02-A | 메가커피 검색 결과 1건 이상 반환 | 0건 반환 |
| 02-B | 컴포즈커피 검색 결과 1건 이상 반환 | 0건 반환 |
| 02-C | 빽다방 검색 결과 1건 이상 반환 | 0건 반환 |
| 02-D | 더벤티 검색 결과 1건 이상 반환 | 0건 반환 |
| 02-E | 매머드커피 검색 결과 1건 이상 반환 | 0건 반환 |
| 02-F | 모든 브랜드 결과가 하나의 목록으로 통합되어 표시되는가 | 특정 브랜드 누락 |
| 02-G | API 응답 상태 코드가 200인가 | 4xx / 5xx 수신 |
| 02-H | 검색 응답 시간 3초 이내인가 | 3초 초과 → WARNING |

**코드 검증 포인트**:
```
- LOW_PRICE_BRANDS 배열에 6개 브랜드 모두 포함 여부
- Promise.allSettled() 사용 여부 (하나 실패해도 나머지 결과 표시)
- API 키가 환경변수(import.meta.env.VITE_KAKAO_REST_API_KEY)에서 로드되는가
- 검색 파라미터: x(경도), y(위도), radius, sort=distance 포함 여부
```

## RULE-03 · 매장 핀 위치 정확도

**언제 적용**: 지도 마커 렌더링 코드 수정 시, 좌표 처리 로직 수정 시

| ID | 검증 내용 | FAIL 조건 |
|----|----------|----------|
| 03-A | 핀 좌표가 API 응답의 x(경도), y(위도) 값을 그대로 사용하는가 | 좌표 변환 오류 |
| 03-B | 줌인 시 핀이 동일한 지리적 위치에 고정되어 있는가 | 핀 위치 이동 |
| 03-C | 줌아웃 시 핀이 동일한 지리적 위치에 고정되어 있는가 | 핀 위치 이동 |
| 03-D | 지도 드래그 시 핀이 지도와 함께 자연스럽게 이동하는가 | 핀이 화면에 고정됨 |
| 03-E | 핀 클릭 시 해당 매장의 이름·주소가 말풍선으로 표시되는가 | 미표시 또는 오정보 |
| 03-F | 핀이 바다, 강, 공원 등 비상식적인 위치에 찍히지 않는가 | 비상식적 위치 |

**코드 검증 포인트**:
```
- 카카오 API 응답: document.x = 경도(lng), document.y = 위도(lat) — 순서 혼동 주의
- new kakao.maps.LatLng(lat, lng) 순서가 올바른가 (y, x 순서)
- 마커 생성 시 map 객체에 정상 바인딩되는가
- 마커 객체가 상태(state/ref)로 관리되어 리렌더링 시 중복 생성되지 않는가
```

<important>
좌표 순서 오류는 핀이 엉뚱한 위치에 찍히는 가장 흔한 버그다.
카카오 API는 x=경도, y=위도를 반환하지만
kakao.maps.LatLng()는 (위도, 경도) 순서를 요구한다.
반드시 new kakao.maps.LatLng(Number(doc.y), Number(doc.x)) 형태로 사용할 것.
</important>

## RULE-04 · 목록 뷰

**언제 적용**: 매장 목록 컴포넌트 수정 시, 정렬·필터 로직 수정 시

| ID | 검증 내용 | FAIL 조건 |
|----|----------|----------|
| 04-A | 목록이 거리 오름차순(가까운 순)으로 정렬되는가 | 거리 역순/무작위 |
| 04-B | 각 카드에 매장명이 표시되는가 | 미표시 |
| 04-C | 각 카드에 거리(m/km)가 표시되는가 | 미표시 |
| 04-D | 각 카드에 주소가 표시되는가 | 미표시 |
| 04-E | 브랜드 필터 선택 시 해당 브랜드만 표시되는가 | 다른 브랜드 혼입 |
| 04-F | 필터 해제 시 전체 브랜드 목록이 복원되는가 | 일부 누락 |
| 04-G | 반경 변경(500m↔1km↔2km) 시 목록이 즉시 갱신되는가 | 갱신 없음 |

**코드 검증 포인트**:
```
- 거리 정렬: sort((a, b) => Number(a.distance) - Number(b.distance))
- API 응답의 distance는 문자열 — Number() 변환 필수
- 필터 로직이 원본 데이터를 변형하지 않고 파생 상태로 처리되는가
- 반경 변경 시 useEffect 또는 이벤트 핸들러가 API 재호출을 트리거하는가
```

## RULE-06 · API 및 네트워크 오류 처리

**언제 적용**: API 호출 코드 수정 시, 에러 핸들링 로직 수정 시, 항상 배포 전

| ID | 검증 내용 | FAIL 조건 |
|----|----------|----------|
| 06-A | 인터넷 끊김 시 오프라인 안내 메시지가 표시되는가 | 빈 화면/무한 로딩 |
| 06-B | API 응답 0건 시 "주변에 매장이 없습니다" 메시지가 표시되는가 | 빈 화면 |
| 06-C | API 오류(4xx/5xx) 시 재시도 버튼이 표시되는가 | 조용한 실패 |
| 06-D | 로딩 중 스피너 또는 스켈레톤 UI가 표시되는가 | 즉각 빈 화면 → WARNING |
| 06-E | API 호출이 5초 초과 시 타임아웃 처리가 되는가 | 무한 로딩 |

**코드 검증 포인트**:
```
- fetch/axios 호출에 AbortController + setTimeout(5000) 타임아웃 적용 여부
- navigator.onLine 체크 또는 online/offline 이벤트 리스너 등록 여부
- 모든 catch 블록에서 사용자에게 상태를 알리는 setState 호출 여부
```
